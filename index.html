bootstrap:表单

.form-control：于input、select、textarea添加类名，增加定制效果；

  1. 在Bootstrap中使用input时也必须添加type类型，为了让控件在各种表单风格中样式不出错，需要添加类名“form-control”；
  2. 文本域<textarea>和原始使用方法一样，设置rows可定义其高度，设置cols可以设置其宽度。但如果textarea元素中添加了类名“form-control”类名，则无需设置cols属性。因为Bootstrap框架中的“form-control”样式的表单控件宽度为100%或auto。
  3. 在form-control基础上：
    .input-sm:让控件比正常大小更小
    .input-lg:让控件比正常大小更大
  4. 焦点状态是通过伪类“:focus”来实现；
  5. 禁用状态：只需要在需要禁用的表单控件上加上“disabled”即可：
    <input class="form-control" type="text" placeholder="表单已禁用，不能输入" disabled>
    如果fieldset设置了disabled属性，整个域都将处于被禁用状态:
    <fieldset disabled></fieldset>
    如果legend中有输入框的话，这个输入框是无法被禁用的;

.form-horizontal：在<form>元素上使用，实现水平表单效果；

在Bootstrap框架中要实现水平表单效果，必须满足以下两个条件：
1、在<form>元素是使用类名“form-horizontal”。
2、配合Bootstrap框架的网格系统。

.form-inline：实现表单在同一行显示；

.checkbox和.radio：来处理复选框、单选按钮与标签的对齐方式。

  1. 不管是checkbox还是radio都使用label包起来了
  2. checkbox连同label标签放置在一个名为“.checkbox”的容器内
  3. radio连同label标签放置在一个名为“.radio”的容器内
  4. 如果checkbox需要水平排列，只需要在label标签上添加“.checkbox-inline”
  5. 如果radio需要水平排列，只需要在label标签上添加“.radio-inline”

表单验证:
验证状态样式,使用的时候只需要在form-group容器上对应添加状态类名:

  1. .has-warning:警告状态（黄色）
  2. .has-error：错误状态（红色）
  3. .has-success：成功状态（绿色）
  4. 相应的icon图标，只需要添加：.has-feedback。在 Bootstrap 的小图标都是使用@font-face来制作。必须在表单中添加了一个 span 元素。

  5. .help-block:将提示信息以块状显示，并且显示在控件底部。

bootstrap样式速查一

.list-unstyled：去除默认表格样式；
.list-inline:水平列表。专注导航100年;
.dl-horizontal:水平定义列表。只有屏幕大于768px的时候，才具有水平定义列表效果；
.lead：强调样式；

.text-muted：提示，使用浅灰色（#999）；
.text-primary：主要，使用蓝色（#428bca）；
.text-success：成功，使用浅绿色(#3c763d)；
.text-info：通知信息，使用浅蓝色（#31708f）；
.text-warning：警告，使用黄色（#8a6d3b）；
.text-danger：危险，使用褐色（#a94442）；

.text-left：左对齐；
.text-center：居中对齐；
.text-right：右对齐；
.text-justify：两端对齐；

代码风格：

  1. 使用<code></code>来显示单行内联代码
  2. 使用<pre></pre>来显示多行块代码
  3. 使用<kbd></kbd>来显示用户输入代码
    （不管使用哪种代码风格，在代码中碰到小于号（<）要使用硬编码“<”来替代，大于号(>)使用“>”来替代。）
    .pre-scrollable：控制代码块区域最大高度为340px，一旦超出这个高度，就会在Y轴出现滚动条；

Bootstrap为表格不同的样式风格提供了不同的类名，主要包括：
.table：基础表格
.table-striped：斑马线表格
.table-bordered：带边框的表格
.table-hover：鼠标悬停高亮的表格
.table-condensed：紧凑型表格
.table-responsive：响应式表格
注意要实现悬浮状态，需要在<table>标签上加入table-hover。
.active：表示当前活动的信息
.success：表示成功或者正确的行为
.info：表示中立的信息或行为
.warning：表示警告，需要特别注意
.danger ：表示危险或者可能是错误的行为

初识JavaScript

JS是解释性语言，源代码放在HTML里。
在<head></head>里说函数的定义、变量的定义。
在<body></body>里是去完成实际的动作。

比如，用JS写一篇文章：。。。

<script onLoad="alert('hi')">
    var hello="Hello"；
    document.write("Hello World");
</script>
  • onLoad=”alert(‘hi’)在打开页面时弹出提示框“hi”；
  • var（varible）定义变量（变量是指在表达式中存放数据的地，变量第一个字符不能是数字），这里有了一个叫做hello的变量，给它赋值“Hello”（初值）；
  • document（这个对象）“.”（运算符）。document做write这个动作。()内是参数，””内是字符串——将Hello World写到HTML中。

初识bootstrap

bootstrap优点：

  • 简单灵活可用于架构流行的用户界面和交互接口的html、css、javascript工具集。
  • 基于html5、css3的bootstrap，具有大量的诱人特性：友好的学习曲线，卓越的兼容性，响应式设计，12列格网，样式向导文档。
  • 自定义JQuery插件，完整的类库，基于Less等。

注意：bootstrap中的js插件依赖于jq。因此jq要在bootstrap之前引用

备注：

  1. <meta http-equiv="X-UA-Compatible"content="IE=edge">在IE运行最新的渲染模式；
  2. <meta name="viewport"content="width=device-width,initial-scale=1">初始化移动浏览显示（让视口的宽度等于设备物理上的分辨率，初始的缩放比例为1，即不缩放。）
  3. 引入css。备注里是为了IE6-7-8兼容HTML5和css3的样式；
  4. js文件放在body最下边。

github for windows 各种failed解决办法

  • 想想自己没有设置git config –global user.name 和 user.email
    在命令行shell输入：

    git config –global user.name “Your Name Here”
    git config –global user.email “your_email@youremail.com”

  • 如果之前设置过，那么：

    a. 把Repository目次中的.git文件夹（隐蔽）的删除
    b. github客户端重新clone分支
    c. 客户端跟新需要上传的工程，重新sync

Hexo探索（一）：安装配置问题与解决

刚听说Hexo的时候，我完全不知道这是什么。什么叫做小白起步、雏鹰起飞……但是，真正用起来，却觉得这真是一个神奇的东西。

  • 用hexo在github上创建个人博客
    推介一个详细的blog：http://www.jianshu.com/p/05289a4bc8b2

  • hexo安装配置问题与解决

    1. 首先，在hexo根文件下右击–Git Bush做好准备；

    2. 问题一：无数次hexo d，皆失败；

      解决：
      hexo init–npm install hexo --save–hexo g–npm install–npm install hexo-deployer-git --save–hexo g–hexo d。

    3. 问题二：提示找不到github；

      解决：
      注意在根目录的 _config.yml 中，最后type=git，不是github。

    4. 问题三:部署时，要输入username和password，而且password打字不显示？！！

      解决：
      用户名是github的注册邮箱，密码直接输入，然后enter就好。（看不见就对了，但一定要输入正确）
